# Test Solver

Локальный инструмент для учебных/разрешённых тестов: запускает отдельный Chrome, читает вопросы через Playwright, ищет ответы по лекциям и сохраняет уникальные вопросы в SQLite + CSV.

## Ограничения

- Инструмент не обходит капчу, прокторинг, антибот-защиту, блокировки DevTools или запреты платформы.
- Автоклик выключен по умолчанию. Для отправки ответов нужен явный флаг `--allow-clicks`.
- Если уверенность ответа низкая или тип вопроса не распознан, инструмент сохраняет вопрос как `needs_review` и не кликает.

## Установка

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -e .
playwright install chromium
```

## Быстрый старт

1. Сложите лекции в папку `lectures/` (`.txt`, `.md`, `.html`, `.pdf`, `.docx`).
2. Импортируйте лекции:

```bash
test-solver ingest lectures --db data/questions.sqlite --csv data/questions.csv
```

3. Запустите отдельный Chrome:

```bash
test-solver chrome --profile .chrome-profile --port 9222
```

4. В открытом Chrome вручную войдите на сайт и откройте тест.
5. В другом терминале выполните диагностику:

```bash
test-solver diagnose --url http://127.0.0.1:9222 --db data/questions.sqlite --csv data/questions.csv
```

6. Для разрешённого автоклика:

```bash
test-solver run --url http://127.0.0.1:9222 --db data/questions.sqlite --csv data/questions.csv --allow-clicks
```

## Команды

- `ingest PATH...` — индексирует локальные лекции.
- `chrome` — запускает отдельный Chrome с remote debugging.
- `diagnose` — читает текущую страницу теста, сохраняет структуру, не кликает.
- `run` — пытается ответить на распознанные вопросы; кликает только с `--allow-clicks`.
- `export` — экспортирует базу вопросов в CSV.

## Проверка на демо-странице

```bash
source .venv/bin/activate
test-solver ingest examples/demo_lecture.txt
test-solver chrome --profile .chrome-profile --port 9222
```

В открывшемся Chrome откройте файл:

```text
file:///Users/x/Documents/решение%20тестов/examples/demo_test.html
```

Затем в другом терминале:

```bash
source .venv/bin/activate
test-solver diagnose --url http://127.0.0.1:9222
test-solver run --url http://127.0.0.1:9222
```

Ожидаемый результат в терминале: вопрос `Что такое HTTP?`, варианты ответа и найденный ответ `Протокол передачи гипертекста`. Без `--allow-clicks` страница не изменится.

Для проверки разметки Moodle откройте:

```text
file:///Users/x/Documents/решение%20тестов/examples/moodle_demo.html
```

И запустите мониторинг:

```bash
test-solver watch --url http://127.0.0.1:9222
```

`watch` держит подключение к Chrome и печатает новый вопрос сразу при переходе по страницам теста. Остановить можно через `Ctrl+C`.

## Moodle

### Рекомендуемый режим: Chrome-расширение

Этот режим работает в обычном Chrome. После установки расширения достаточно обновить страницу теста на `https://edu.gubkin.ru/`.

1. Запустите локальный backend:

```bash
cd "/Users/x/Documents/решение тестов"
source .venv/bin/activate
test-solver serve
```

2. Откройте Chrome и перейдите:

```text
chrome://extensions/
```

3. Включите `Режим разработчика`.
4. Нажмите `Загрузить распакованное расширение`.
5. Выберите папку:

```text
/Users/x/Documents/решение тестов/chrome_extension
```

6. Откройте или обновите страницу теста на `https://edu.gubkin.ru/`.

На странице появится панель `Test Solver`. Она показывает найденный вопрос, ответ, уверенность и источник. База и CSV обновляются автоматически.

Панель можно перетаскивать мышью за заголовок и сворачивать кнопкой `−/+`. Позиция сохраняется в Chrome.

Если открыть страницу просмотра результатов Moodle после завершения попытки, расширение ищет блоки правильных ответов Moodle и сохраняет только подтверждённые правильные ответы в базу с источником `moodle-review`.

Если локальная база и лекции не дали ответ, backend пробует расширенный веб-поиск и помечает такой результат источником `web-search-v2`. Он строит несколько поисковых запросов, читает найденные страницы, сравнивает варианты ответа с релевантными фрагментами и кэширует результат в SQLite. Это вспомогательная подсказка для проверки качества тестов, а не подтверждённый ответ Moodle.

Настройки веб-поиска backend:

```bash
test-solver serve --web-search --web-timeout 8 --web-max-pages 5 --web-cache-ttl 86400
```

Отключить интернет-поиск можно так:

```bash
test-solver serve --no-web-search
```

В popup расширения можно выключить отображение окна с ответом переключателем `Показывать окно с ответом`. Это скрывает overlay, но не выключает расширение полностью. Полное выключение делается кнопкой `Выключить` или горячей клавишей.

На странице review расширение отдельно считает статусы Moodle: `Верно`, `Неверно`, `Частично`. Если Moodle не показывает правильный ответ для неверного вопроса, такой вопрос отображается как `needs_review` и не сохраняется как правильный.

Расширение можно выключить:

- через иконку расширения в Chrome: кнопка `Включить/Выключить`;
- горячей клавишей `Alt+Shift+S`;
- изменить горячую клавишу можно на странице `chrome://extensions/shortcuts`.

### Перенос backend на сервер

Расширение не привязано к локальному компьютеру. В popup расширения есть поле `Backend URL`.

Локально по умолчанию:

```text
http://127.0.0.1:8765
```

На выделенном сервере запускайте backend на внешнем интерфейсе:

```bash
test-solver serve --host 0.0.0.0 --port 8765
```

Для реального использования на разных устройствах лучше поставить reverse proxy с HTTPS, например:

```text
https://solver.example.com
```

После этого в popup расширения укажите этот адрес в `Backend URL` и нажмите `Сохранить backend`. Backend должен иметь доступ к общей базе `data/questions.sqlite`, импортированным лекциям и CSV. Сервер уже отдаёт CORS-заголовки для запросов расширения.

### Альтернативный режим: Playwright/remote debugging

1. Если нужно работать в обычном профиле Chrome, полностью закройте Chrome через `Cmd+Q`.
2. Запустите Chrome с debug-портом и обычным профилем:

```bash
test-solver chrome --default-profile --port 9222
```

3. В этом Chrome войдите в Moodle и откройте попытку теста.
4. Проверьте, что вкладка видна:

```bash
test-solver pages --url http://127.0.0.1:9222
```

5. Запустите наблюдение:

```bash
test-solver watch --url http://127.0.0.1:9222 --page-url moodle
```

Если вкладка уже была открыта в Chrome, который запущен без `--remote-debugging-port`, подключиться к ней задним числом нельзя. Нужно перезапустить Chrome командой выше; обычная сессия и логин сохраняются в стандартном профиле.

Альтернативный изолированный режим:

```bash
test-solver chrome --profile .chrome-profile --port 9222
```

Затем в этом Chrome войдите в Moodle и откройте попытку теста.

Инструмент читает Moodle-блоки `.que`, текст вопроса из `.qtext`, варианты из `.answer`, сохраняет новые вопросы в `data/questions.csv` и показывает найденный ответ в терминале. Автоклик возможен только с `--allow-clicks`.

## Данные

SQLite хранит:

- `lectures` — документы и извлечённый текст.
- `questions` — уникальные вопросы, варианты, найденные ответы, статус и источник.
- `attempts` — история распознавания/ответов.
